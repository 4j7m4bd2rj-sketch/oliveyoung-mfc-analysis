# 🫙 OliveYoung Delivery Optimization Simulator

> 올리브영 멀티센터 주문분배 로직 시뮬레이터  
> 올리브영 테크블로그 공개 수치 기반으로 설계된 시뮬레이션 프로젝트

---

## 📌 프로젝트 배경

올리브영 테크블로그(2026.03.06)에서 공개한 배송최적화 시스템 구축기를 바탕으로,  
멀티센터 주문분배 로직을 직접 구현하고 **쿠팡식 접근 방식과 구조적으로 비교**한 프로젝트입니다.

**핵심 질문:**  
> 피크타임(올영세일)에 올리브영 방식과 쿠팡식 방식 중 어느 쪽이 취소율과 리드타임에서 유리한가?

---

## 🏗️ 분배 로직 구조

### 올리브영 방식 (테크블로그 4단계)
```
1단계. SKU 재고 보유 센터 확인 (양지 / 경산)
2단계. CAPA 여유 확인 → 초과 시 취소 (팬텀 재고)
3단계. 우편번호 기준 권역 매칭 → 최적 센터 우선순위
4단계. 최적 센터 확정 → 출하지시
```

### 쿠팡식 방식 (구조적 비교)
```
- 주문 즉시 재고 선점 → 팬텀 재고 없음
- 단일 대형 FC 중심 출고
- CAPA 초과 시 취소 없이 지연 처리
```

---

## 💡 주요 인사이트

| 지표 | 올리브영 방식 | 쿠팡식 방식 |
|------|------------|-----------|
| 취소 원인 | CAPA 초과 (팬텀 재고) | 재고 소진 |
| 리드타임 | 권역 최적화로 단축 | 단일 FC → 균일하나 더 길음 |
| 강점 | 비수도권 리드타임 우위 | 취소율 낮음 (재고 선점) |
| 개선 포인트 | 재고 선제 배치 고도화 | 비수도권 배송 속도 |

**핵심 발견:**  
올리브영 방식은 권역 기반 분배로 리드타임에서 우위를 갖지만,  
피크타임 CAPA 초과 시 팬텀 재고로 인한 취소가 발생함.  
"결제 후 취소 제로화"를 위해서는 **재고 선제 배치 + 출고 가능 재고 우선 확정 로직** 고도화가 핵심.
